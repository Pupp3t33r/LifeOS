wallet · Wishlist — the wants backlog (Money ADR-0022 · 0034)

Everything you're wishing for

A flat list of the things you want — the backlog Plan · Board pulls from but never let you edit. Here you add, price, and remove wants; find them by category and search. Each one shows how far it's travelled: from a wish, to planned on a month or in a payment plan, to bought.

No grouping — just wants. One item per thing you want; category filters and search are the way around it (no folders, no bundles). Status is read here, not set: a want turns Planned when you drop it on a month on the Board, Financed when it rides inside a payment plan, and Bought when it's paid (Idle · Planned · Financed · Bought — the as-built projection, Money ADR-0034, derived from period + plan events, never hand-edited; shown as Wishing · Planned · Paying off · Bought). This surface owns the desire side only: name, estimate (optional — a want may have no known price yet, ADR-0030), notes, and the one-time / repeat flag. Bought wants are hidden by default (tucked into a collapsed row); the default view is Active — everything still in play. Linking a want to a domain object (ADR-0030's ExternalRef) is out of scope for now — so the manual picker offers only ordinary user categories. The system/domain categories (Books · Board Games · Video Games, ADR-0024) are link-derived: a want lands in one only by being linked to that domain object (auto-categorized from the ref), so they're not hand-pickable here and light up when linking ships. Nav slot is deliberately open — ADR-0005 §9 defers it to the app-wide IA review; drawn here as candidate (b), pushed from the Board tray, so it claims no bottom-bar slot yet. ⚠️ Open: how a Repeat want shows its status is undecided — the track is a one-way journey, but a Repeat want restocks (never terminal); pinned at Wishing here as a placeholder, to be resolved before build.
