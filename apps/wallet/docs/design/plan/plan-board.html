wallet · Plan · Board view — the try-on board

Plan by dragging wants into months

The other side of the List (direction A). Your wishlist sits in a tray; the months ahead are columns. Drag a want onto a month to try it on — the month's total shifts so you feel the weight before you commit. Drag it to another month to reschedule, or back to the tray to drop the idea. The drop is the commit — no dialog, no confirm. One gesture, one page.

Same tab, second view. Board is a List ⇄ Board toggle on Plan, not a separate destination — planning stays one place. A drop files a planned purchase on that period (ADR-0018, which already allows planning onto future months); dragging between months re-files it; the tray is your Wishlist (ADR-0022 items). Each column shows the month's committed base (Ongoing + plan payments) so the wants you're trying on read as weight on top of what's already fixed.

Phone drags for real. On drag-start the months collapse to header rows (name + balance) and the wishlist collapses to a single cancel bar — so every month in the window becomes a visible drop target and there's a clear "drop to cancel." On drop, everything expands back to the shortened resting view. No picker panel, no dialog. Two phone frames below show resting and mid-drag.

Model, pinned down. Months run from this month forward — all twelve for a future year; pick the year with ‹ 2026 ›, no window pager, columns scroll when long. Wants are once (fridge, tires — leaves the wishlist when planned, returns if removed; a move relocates the one instance, never duplicates) or ↻ reusable (coffee, tea — stays, plan any month). A want financed by a payment plan isn't a draggable card — it's already committed, so it shows only as each month's fixed weight. A planned single buy has no day: the month is its schedule (target period).
