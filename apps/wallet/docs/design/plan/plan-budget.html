wallet · Plan · Budget view — spending limits & savings target

Set the shape — limits and a target

The third view of Plan, alongside List and Board. Here you set a spending limit per category and a savings target — the levers that define what a good month looks like. You set them once; Home tracks the actuals against them, period by period. This is the destination Home's budget strip “Manage” link has been pointing at.

The list is a selection, not a creation. You add / remove which existing categories are tracked for budget compliance — you don't make new categories here (category management comes later). A category that's not on the list isn't tracked; its spend pools into Other in budget stats — a residual bucket with no limit. So the limit list is opt-in per category.

Furthest from today's model — build last. Limits and target are period-agnostic templates you set here; the live spent / limit tracking belongs to Home, not this view. The right-hand “Your month” readout is plain arithmetic on your own levers (expected income − limits − target = free), not the composed forecast: MonthProjection / MonthlyReview (ADR-0007) is Accepted but unbuilt, and neither budgets nor a target exist in the money service yet. So of the three views this is the largest backend lift.
